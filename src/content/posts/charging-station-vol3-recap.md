## 聰電站是什麼？

聰電站是 WPORT 在桃園定期舉辦的**免費實體活動系列**，每一場聚焦一個主題，不講廢話、全程動手做，目標是讓大家帶著看得見的成果回家。想了解聰電站的完整介紹與過往活動，可以看[這篇說明](/blog/posts/charging-station/)。

Vol.3 的主題是**「沒有資科背景的人也能實作的 AI 課程」**，不需要任何程式基礎，全程動手操作。由 Eric（WPORT PM）與 David（後端工程師）帶課，三個小時帶你走完從環境建置到網站上線的完整流程：

- **AI IDE**：認識 Cursor、Codex、Kiro、Antigravity 四款免費工具，了解為什麼工作場景要從 ChatGPT 聊天框搬到 IDE
- **GitHub Fork**：把課程 Repo 複製到自己帳號，建立第一個屬於自己的 AI 專案
- **Obsidian 知識譜系**：把個人經歷、技能整理成 AI 可讀的結構化資料
- **Skill 反問法**：讓 AI 幫你想內容，而不只是幫你寫
- **CLI 撈職缺 × 履歷客製**：一行指令抓桃園 PM 職缺，自動產出三份客製履歷
- **Vercel 部署**：個人網站一行指令上線

## 這次的課程在教什麼？

三個小時的桃園 AI 課程，我們把整條 2026 主流的 AI 工作流走一遍，分成四個階段：

**階段 A：環境建置 × AI IDE 選擇**
從 GitHub Fork 課程 Repo 開始，認識 Cursor、Codex、Kiro、Antigravity 四款免費 AI IDE，理解為什麼要從 ChatGPT 聊天框搬到 IDE 工作環境。

**階段 B：用 AI 讀懂陌生專案**
把 README 拖進 AI IDE 對話區，3 分鐘內讓 AI 幫你讀懂整個 Repo 的目的與結構，建立「先讀 README」的職業習慣。

**階段 C：Obsidian 知識譜系 × 個人網站生成**
用 Obsidian 把自己的經歷、技能、作品整理成結構化資料，再透過 Skill 反問法讓 AI 幫你挖內容，最後用 local path 一鍵生成個人網站 HTML。

**階段 D：Prompt → Skill → CLI × MCP 自動化管線**
把有效的 Prompt 升級成可重用的 Skill，用 WPORT CLI 撈桃園職缺、客製三份履歷，再用 Vercel CLI 一行指令把網站部署上線。

| 時間段 | 課程單元 | 使用工具 |
| :--- | :--- | :--- |
| 0 – 40 分鐘 | AI IDE vs Chatbot、GitHub Fork | GitHub、Cursor / Codex / Kiro / Antigravity |
| 40 – 90 分鐘 | 用 AI 讀懂陌生 Repo | AI IDE、README.md |
| 90 – 150 分鐘 | Obsidian 知識譜系 × Skill 反問 × 網站生成 | Obsidian、Skill、Local Path |
| 150 – 180 分鐘 | CLI 撈職缺、履歷客製、Vercel 部署 | WPORT CLI、Vercel CLI、MCP、Skill |

## 活動現場

![桃園 AI 課程「聰電站 Vol.3」現場，講師在大螢幕前帶著學員實作](https://res.cloudinary.com/dyebbsckc/image/upload/f_auto,q_auto:good,w_1200,c_limit/wport-blog/charging-station-instructor-demo.jpg)

這場桃園 AI 課程全程都在動手操作，不是只聽講。大家邊做邊問，遇到卡關就當場拆解，通常問題都能在現場被解掉。

還有一段讓我特別有感的畫面。**開南大學就在桃園蘆竹，離我們活動場地騎車只要幾分鐘**，這次剛好有兩位我之前在開南大學授課帶過的學生也主動報名來參加。看到學生在正式課堂之外，還願意騎車跑來繼續學習 AI IDE、CLI 這些新工具，真的是身為講師最欣慰的畫面之一。這也正是我們一直想做的事：**讓桃園在地的學生、上班族，不用一直往台北跑，也能接觸到最新的 AI 應用**。

其中一位平常沒有寫程式經驗的學員，在第二階段結束時成功把 Fork 的專案跑起來，自己也很驚訝地說「原來這樣就可以了」。另外也有學員在第三階段把零散經歷整理成知識譜系，第一次看到 AI 產出「真的像自己的」網站內容。

三個小時下來，大家完成的不只是一份課堂筆記，而是一條能繼續延伸的實作路線。

更重要的是，大家離開前都有同一種感受：原來 AI 不只是拿來問答案，也可以拿來整理自己、產出作品、支持求職。這個轉變對初學者來說，比學會某一個指令更關鍵。
