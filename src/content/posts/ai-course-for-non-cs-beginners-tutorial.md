## 單元一：你習慣的是聊天框，但主角其實是 IDE

### 大多數人用 AI 的方式，其實很浪費

很多人開始用 AI 的方式是這樣的：打開 ChatGPT，輸入問題，看回答，然後關掉。

這種方式當然有用，但它有一個根本的限制：**對話結束之後，什麼都不會留下來。**

AI 給你的答案、你整理好的思路、你反覆確認過的做法，全部都只存在那個聊天視窗裡。下次再遇到同樣的問題，你還是要重新問一遍。

這一單元要介紹的，是一個更好的工作方式。

### 什麼是 IDE？

IDE 是 Integrated Development Environment（整合開發環境）的縮寫。

聽起來很技術，但你可以把它理解成**一個比記事本更強大的工作空間**，不只是讓你輸入文字，而是讓你管理整個專案、執行程式、看到結果，全部在同一個地方完成。

過去 IDE 是工程師的工具。但 2026 年之後，這個情況正在改變。

### 2026 新世代 AI 優先的 IDE

現在有幾個 IDE 工具，把 AI 直接整合進工作環境裡，讓非工程師也能用：

<img src="https://res.cloudinary.com/xyudkke9/image/upload/f_auto,q_auto/截圖_2026-07-02_上午11.23.32_wbcgxq" alt="2026 新世代 AI 優先 IDE 工具介紹簡報" style="box-shadow: 0 4px 24px rgba(0,0,0,0.12); border-radius: 12px;" />

#### Cursor

目前最多人使用的 AI IDE，搭配 Claude 生態系。你可以直接在裡面對話、讓 AI 修改你的檔案、解釋程式碼，而且它會記得你整個專案的結構，不只是單一對話。

#### Kiro IDE

支援 Spec-First 開發方式的工具。你先用自然語言描述你想做什麼，Kiro 會幫你拆解成具體的任務清單，再一步一步協助你完成。

#### Codex IDE

OpenAI 推出的 AI 程式助理環境，適合想要直接用英文描述需求、讓 AI 生成可執行內容的用戶。

#### Antigravity 2.0

整合 AI 的全新開發體驗，特別適合從零開始建立專案的初學者。

### 為什麼非工程師也要了解這些工具？

因為這些工具正在改變「工作」的定義。

不管你是做行銷、做設計、做業務，還是做管理，你的工作裡都有大量重複性的文字處理、資料整理、流程自動化需求。這些工具可以幫你把 AI 的能力真正帶進你的工作流程，而不只是用來回答問題。

## 單元二：Fork 課程 Repo，認識你的專案

這個單元是很多學員覺得最有成就感的部分，因為你會在這裡第一次讓自己的「專案」跑起來。

### 什麼是 Fork？什麼是 Repo？

**Repo（Repository）** 是「專案儲存庫」的意思。你可以把它想像成一個資料夾，但這個資料夾除了放你的檔案之外，還會記錄每一次的修改歷史，讓你隨時可以回到之前的版本。

**Fork** 是「複製一份到自己的帳號」的意思。就像是你在圖書館看到一本書，把它影印一份帶回家，以後可以在上面做筆記、畫線、改寫，但原版不會被影響。

這個單元讓學員 Fork 了課程的 GitHub Repo，把課程的範例專案複製到自己的帳號下，然後在本機上跑起來。

### 步驟一：建立 GitHub 帳號

<img src="https://res.cloudinary.com/xyudkke9/image/upload/f_auto,q_auto/截圖_2026-07-02_上午11.41.12_a4xa2y" alt="建立 GitHub 帳號步驟說明簡報" style="box-shadow: 0 4px 24px rgba(0,0,0,0.12); border-radius: 12px;" />

如果你還沒有 GitHub 帳號，先到 [github.com](https://github.com) 註冊一個。

GitHub 是目前最主流的程式碼托管平台，也是工程師社群最活躍的地方。即使你不寫程式，有一個 GitHub 帳號在某些工作場景也會讓你看起來更懂行。

### 步驟二：Fork 課程 Repo

找到課程的 GitHub Repo，點擊右上角的「Fork」按鈕，選擇要複製到的帳號（就是你自己），然後等它複製完成。

完成之後，你的 GitHub 頁面裡就會出現一個跟課程一模一樣的 Repo，但它是屬於你的。

### 步驟三：安裝環境，在本機跑起來

這裡需要安裝幾個工具，包含 Node.js（一個讓電腦可以執行 JavaScript 的環境）和 Git（讓你可以把 GitHub 上的 Repo 下載到本機）。

安裝完成之後，在你的 IDE 裡打開終端機，輸入幾行指令，你的專案就會在本機跑起來，可以用瀏覽器看到結果。

第一次看到「本機跑起來」的畫面，很多人的反應是：「原來這樣就可以了啊。」

這個瞬間很重要，因為它打破了一個迷思：做出一個可以執行的東西，沒有你想像的那麼難。

### 步驟四：讓 AI 幫你讀懂程式碼

有了 IDE 之後，你不需要一行一行看懂所有程式碼。你可以直接問 AI：「這個檔案在做什麼？」、「這行程式碼的意思是？」、「我想改變這個部分，應該怎麼做？」

AI 會用你聽得懂的語言解釋，然後告訴你要改哪裡、改什麼。你只需要確認它說的對不對，然後按下執行。

## 單元三：講完就消失，還是固化下來？

這是整個課程裡最容易被忽略、但長期來看影響最大的一個單元。

### AI 給的好答案，你有沒有留下來？

想一個場景：你花了 20 分鐘跟 AI 來回溝通，終於得到一個非常精準的回答，或者一個非常好用的 Prompt。然後你關掉視窗。下次遇到同樣的需求，你還是要重新花 20 分鐘。

這就是「沒有固化」的問題。大多數人用 AI 的方式，是把它當作即問即答的工具，但這樣等於每次都在重頭來過。

<img src="https://res.cloudinary.com/xyudkke9/image/upload/f_auto,q_auto/截圖_2026-07-02_上午11.25.30_mfkbjx" alt="Prompt 固化概念說明簡報" style="box-shadow: 0 4px 24px rgba(0,0,0,0.12); border-radius: 12px;" />

### 什麼是 Prompt 固化？

固化不是把 Prompt 複製貼上存到記事本。它是一個更系統化的做法：

**把有效的 Prompt 存成可重複使用的模板，嵌入你的工作流程裡。**

這樣每次有需要的時候，你不需要重新想怎麼問，直接用已經驗證過的 Prompt，結果更穩定，效率也更高。

### 如何建立你的 Prompt 庫？

**第一步：記錄每一個「這個 Prompt 真的很好用」的瞬間**

當你發現某個問法特別有效，馬上把它存起來。不要依賴記憶，記憶不可靠。

**第二步：標記使用情境**

光存 Prompt 還不夠，你要標記它適合用在什麼情況。例如：「寫文章開頭」、「整理會議記錄」、「回覆客戶信件」。

**第三步：定期回顧和更新**

AI 工具在進化，你的需求也在改變。每隔一段時間回顧你的 Prompt 庫，把不再好用的刪掉，把新發現的加進來。

### 把 Prompt 嵌入工作環境

更進一步的做法，是直接把固化的 Prompt 放進你的 IDE 工作環境裡。

在 Cursor 等 AI IDE 中，你可以建立「規則檔案」，讓 AI 在每次工作的時候自動套用你設定好的方式和風格。這樣你不需要每次都重新說明你的偏好，AI 已經知道了。

## 三個單元學完，你應該可以做到什麼？

如果你把這三個單元都跟著做了一遍，你現在應該可以：

- 用 Cursor 或類似的 AI IDE 開啟一個專案
- 看懂一個 GitHub Repo 的基本結構，Fork 一份到自己的帳號
- 在本機把一個專案跑起來
- 用 AI 讀懂你看不懂的程式碼，並做出基本的修改
- 建立自己的 Prompt 庫，把有效的問法保存下來重複使用

這些不是進階技能，這是 2026 年的基本配備。
